Functions

  • Set of commands for doing something, that may or may not take some input and that generates some output.
  • Called by typing the function name, followed by parentheses which may contain arguments that are passed to the function.
  • Entering the function name without parentheses returns the function's code. (Try this for one of the functions you have learned.)
  • Creating a function involves specifying arguments (if any), listing the expressions to be evaluated, and specifying the value(s) to be returned

Functions

Writing your own function -- example: 

			my_super_function <- function(a, b=10){
				x1 <- a*b
				x2 <- x1-45
				x3 <- x2^2
				return(x3)
			}

Basic Organization and Workflow

  • Create a directory for each data analysis project you are working on .
  • Use the RStudio projects to automatically start R with the correct working directory for the project; or use setwd() to set the desired working directory.
  • Use file names that mean something.
  • Decide if you want to keep the .RData file when closing sessions. (I usually do not keep this.).

Packages

  • Install using install.packages() function.
  • Load using library() function or require()function.

Packages

Install then load the Hmisc and tidyverse package


			install.packages(c("Hmisc", "tidyverse"))
			library(Hmisc)
			library(tidyverse)
